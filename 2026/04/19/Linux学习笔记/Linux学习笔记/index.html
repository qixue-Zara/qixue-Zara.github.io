跟着黑马程序员学完Linux，整理了全套核心知识点，涵盖课程重点、实操命令、必背考点，适合新手回顾复盘，也可作为日常开发备查笔记，纯干货无冗余～

一、Linux基础认知

1.1 什么是Linux

  • Linux是一款开源、免费、多用户、多任务的操作系统，核心优势是稳定、安全、占用资源少

  • 与Windows的区别：Windows面向桌面用户，Linux面向服务器（占服务器领域90%以上市场）

  • 常见发行版（黑马重点提及）：CentOS（企业级首选，稳定）、Ubuntu（桌面/服务器通用，易用）、RedHat（收费，企业级）、Debian

1.2 Linux核心思想

✅ 一切皆文件：无论是硬件（CPU、硬盘）、软件、配置，在Linux中都以文件形式存在

✅ 命令行优先：Linux的核心操作是命令行，图形界面仅为辅助（服务器一般无图形界面）

二、Linux文件系统结构（必背）

Linux的文件系统是树形结构，所有文件/目录都挂载在根目录 / 下，黑马强调以下目录必须记牢：

  • /：根目录，所有文件的顶层目录，不可删除

  • /root：超级用户（root）的家目录，只有root能访问

  • /home/用户名：普通用户的家目录（如 /home/test），普通用户只能操作自己的家目录

  • /etc：系统核心配置文件目录（重中之重），如网络配置、服务配置

  • /bin、/usr/bin：系统可执行命令目录（如 ls、cd 等常用命令）

  • /var：动态文件目录，存放日志（/var/log）、缓存、临时数据

  • /tmp：临时文件目录，重启系统后文件会清空

  • /usr：用户程序目录，存放软件安装包、库文件

  • /boot：系统启动目录，存放启动相关文件（不可删除）

三、Linux常用基础命令（黑马实操重点）

所有命令均为黑马课程高频实操命令，标注 必背 的是日常开发最常用的，建议熟练掌握。

3.1 文件/目录操作命令（必背）

命令 作用 常用示例
ls 查看当前目录下的文件/目录 ls（简单查看）、ls \-l（详细查看，简写 ll）
cd 切换目录 cd /home（切换到/home）、cd \.\.（返回上一级）、cd \~（回到家目录）
pwd 显示当前所在路径（必背） pwd
mkdir 创建目录 mkdir test（创建单个目录）、mkdir \-p a/b/c（创建多级目录）
touch 创建空文件 touch test\.txt
cp 复制文件/目录 cp test\.txt /home（复制文件）、cp \-r a /home（复制目录，需加-r）
mv 移动/重命名文件/目录 mv test\.txt /home（移动）、mv test\.txt test1\.txt（重命名）
rm 删除文件/目录（慎用） rm test\.txt（删除文件）、rm \-rf test（强制删除目录，必背，谨慎使用）

3.2 文件查看命令（必背）

  • cat 文件名：查看文件全部内容（适合小文件），示例：cat /etc/hosts

  • more 文件名：分页查看文件，按 space 翻页，q 退出

  • less 文件名：灵活翻页查看，按 ↑↓ 滚动，q 退出（适合大文件，必背）

  • head \-n 数字 文件名：查看文件前n行，示例：head \-10 /var/log/messages（查看前10行日志）

  • tail \-n 数字 文件名：查看文件后n行，示例：tail \-5 test\.txt

  • tail \-f 文件名：实时查看文件变化（重点，常用于查看日志），示例：tail \-f /var/log/nginx/access\.log

3.3 搜索与查找命令

  • find 路径 \-name 文件名：按文件名查找文件/目录（必背），示例：find / \-name test\.txt（从根目录查找test.txt）

  • grep \&\#34;关键词\&\#34; 文件名：过滤文件内容，查找包含关键词的行（必背），示例：grep \&\#34;error\&\#34; /var/log/messages（查找日志中的错误信息）

  • grep \-n \&\#34;关键词\&\#34; 文件名：显示匹配行的行号

3.4 权限操作命令（黑马重点，必背）

Linux中每个文件/目录都有3种权限：读（r）、写（w）、执行（x），对应3类用户：所有者（u）、所属组（g）、其他用户（o）

  • 权限数值：r=4、w=2、x=1，权限组合为数值之和（如 rwx=7、rw-=6、r--=4）

  • chmod 权限值 文件名：修改文件/目录权限（必背），示例：chmod 755 test\.txt（所有者rwx，其他用户r--x）

  • chown 用户名:组名 文件名：修改文件/目录的所有者和所属组，示例：chown root:root test\.txt（修改为root所有）

常用权限：755（文件可执行、目录可访问）、644（普通文件，只读）

3.5 进程与服务命令（必背）

  • ps \-ef：查看所有进程（必背），示例：ps \-ef \| grep nginx（查看nginx进程）

  • top：实时监控进程占用（CPU、内存），按 q 退出

  • kill \-9 进程ID（PID）：强制终止进程（必背），示例：kill \-9 1234（终止PID为1234的进程）

  • systemctl start 服务名：启动服务，示例：systemctl start nginx

  • systemctl stop 服务名：停止服务

  • systemctl restart 服务名：重启服务（必背）

  • systemctl enable 服务名：设置服务开机自启（必背）

  • systemctl status 服务名：查看服务运行状态

3.6 网络相关命令（必背）

  • ifconfig / ip addr：查看服务器IP地址（必背），CentOS7+推荐用 ip addr

  • ping 目标IP/域名：测试网络连通性，示例：ping www\.baidu\.com

  • netstat \-ntlp：查看端口占用情况（必背），示例：netstat \-ntlp \| grep 80（查看80端口占用）

  • hostname：查看服务器主机名

3.7 压缩与解压命令（必背）

Linux中最常用的压缩格式是 \.tar\.gz，黑马重点讲解：

  • 打包压缩：tar \-zcvf 压缩包名\.tar\.gz 要压缩的文件/目录，示例：tar \-zcvf test\.tar\.gz test/

  • 解压：tar \-zxvf 压缩包名\.tar\.gz（解压到当前目录），示例：tar \-zxvf test\.tar\.gz

  • 解压到指定目录：tar \-zxvf 压缩包名\.tar\.gz \-C 目标目录，示例：tar \-zxvf test\.tar\.gz \-C /home

3.8 用户与组命令

  • useradd 用户名：创建普通用户，示例：useradd test

  • passwd 用户名：设置用户密码（必背），示例：passwd test（输入密码时不显示）

  • su 用户名：切换用户，示例：su test（从root切换到test）、su \-（切换到root）

  • sudo：普通用户临时提权（执行root权限命令），需配置sudoers文件

  • userdel \-r 用户名：删除用户（-r 同时删除家目录）

四、Vim编辑器（黑马必会，必背）

Vim是Linux中默认的文本编辑器，用于修改配置文件、编写脚本，核心是3种模式切换，黑马反复强调：

4.1 三种模式及切换

  • 命令模式（默认模式）：打开文件后默认进入，用于执行编辑命令（如删除、撤销）

  • 输入模式：用于输入文本，从命令模式按 i 进入（i=insert）

  • 末行模式：用于保存、退出，从命令模式按 : 进入

  • 模式切换：输入模式 → 命令模式（按 ESC）；命令模式 → 末行模式（按 :）

4.2 常用Vim命令（必背）

  • 命令模式：

    • dd：删除当前行（必背）

    • ndd：删除n行（如 3dd 删除3行）

    • u：撤销上一步操作（必背）

    • yy：复制当前行

    • nyy：复制n行

    • p：粘贴复制的内容（必背）

    • shift \+ g：跳转到文件末尾

    • gg：跳转到文件开头

  • 末行模式（必背）：

    • :w：保存文件

    • :q：退出Vim（文件未修改时）

    • :wq：保存并退出（必背）

    • :q\!：强制退出，不保存修改（必背）

    • :set nu：显示行号

    • :set nonu：隐藏行号

常用场景：修改配置文件 → vim /etc/profile → 按 i编辑 → 按 ESC → :wq 保存退出

五、Shell脚本基础（黑马实操拓展）

Shell是Linux的命令解释器，Shell脚本（.sh）用于自动化执行一系列命令，黑马讲解基础脚本编写：

5.1 脚本规范

  • 脚本文件后缀为 \.sh（如test\.sh）

  • 脚本第一行必须写 \#\!/bin/bash（指定解释器）

  • 执行脚本：bash test\.sh 或 \./test\.sh（需给脚本加执行权限 chmod 755 test\.sh）

5.2 基础脚本示例（黑马演示案例）

5.3 脚本核心语法（基础）

  • 变量：name=\&\#34;test\&\#34;，使用时 echo $name

  • if判断：
    if \[ $name == \&\#34;test\&\#34; \]; then echo \&\#34;正确\&\#34; else echo \&\#34;错误\&\#34; fi

  • for循环：
    for i in 1 2 3 4 5; do echo $i done

常用场景：日志备份、服务重启、批量执行命令

六、Linux软件安装方式（黑马重点）

Linux软件安装有3种常用方式，黑马按“易用性”排序讲解：

6.1 在线安装（最常用，必背）

  • CentOS/RHEL：使用 yum 命令，自动解决依赖

    • 安装：yum install 软件名 \-y（-y 自动确认），示例：yum install nginx \-y

    • 卸载：yum remove 软件名 \-y

    • 更新：yum update 软件名

  • Ubuntu/Debian：使用 apt\-get 命令

    • 安装：apt\-get install 软件名 \-y

    • 卸载：apt\-get remove 软件名 \-y

6.2 rpm包安装（手动安装，适合无网络）

rpm是Linux的软件包格式，需手动下载rpm包，示例：

  • 安装：rpm \-ivh 软件包\.rpm（ivh：安装、显示进度、显示详细信息）

  • 卸载：rpm \-e 软件名

  • 缺点：需手动解决依赖关系，不如yum方便

6.3 源码编译安装（最灵活，适合定制）

步骤（黑马标准流程）：

  1. 下载源码包（如 wget 源码包地址）

  2. 解压源码包：tar \-zxvf 源码包\.tar\.gz

  3. 进入解压目录：cd 源码目录

  4. 配置：\./configure（可指定安装路径）

  5. 编译：make

  6. 安装：make install

常用场景：安装特定版本的软件（如自定义nginx安装路径）

七、防火墙与端口配置（黑马重点，必背）

Linux防火墙用于控制端口访问，黑马重点讲解CentOS7+的 firewalld（替代旧版iptables）：

7.1 firewalld常用命令

  • systemctl start firewalld：启动防火墙

  • systemctl stop firewalld：停止防火墙（测试环境常用）

  • systemctl enable firewalld：防火墙开机自启

  • firewall\-cmd \-\-list\-ports：查看已开放的端口

  • firewall\-cmd \-\-add\-port=端口号/tcp \-\-permanent：永久开放端口（必背），示例：firewall\-cmd \-\-add\-port=80/tcp \-\-permanent

  • firewall\-cmd \-\-reload：重新加载防火墙（开放端口后必须执行，必背）

  • firewall\-cmd \-\-remove\-port=端口号/tcp \-\-permanent：永久关闭端口

注意：云服务器（如阿里云、腾讯云）还需在控制台开放对应端口，否则防火墙开放也无法访问。

八、学习总结与注意事项

8.1 核心重点回顾

  • 必背：文件系统结构、常用命令（ls、cd、pwd、rm、chmod、systemctl等）

  • 必会：Vim编辑器（三种模式、保存退出）、防火墙配置、软件安装

  • 常用：进程查看、日志查看、端口查看

8.2 黑马强调的注意事项

  • 慎用 rm \-rf /（删除根目录，会导致系统崩溃）

  • 修改配置文件前，建议备份（如 cp /etc/profile /etc/profile\.bak）

  • 执行命令时，注意用户权限（普通用户无法修改系统配置，需用root或sudo）

  • 服务器尽量关闭不必要的服务，开放必要的端口，提升安全性
