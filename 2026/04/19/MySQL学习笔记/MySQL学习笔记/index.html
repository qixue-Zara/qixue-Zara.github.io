MySQL基础知识点总结（至事务部分完结）

本文总结MySQL基础部分核心知识点，涵盖从MySQL入门到事务的全部基础内容，适合新手入门梳理，适配个人博客呈现，知识点详细且逻辑连贯，重点内容加粗标注，所有sql语句均使用小写，贴合日常使用习惯。

一、MySQL入门基础

1.1 MySQL简介

  • mysql是一款开源的关系型数据库管理系统（rdbms），由oracle公司维护，支持多平台（windows、linux、mac）。

  • 核心特点：体积小、速度快、开源免费、支持多用户多线程、兼容性强，广泛应用于中小型网站、后台系统的数据存储。

  • mysql的核心组件：数据库（database）、表（table）、字段（field）、索引（index）、sql语句，数据以表格形式组织，遵循关系模型。

1.2 MySQL安装与环境配置（windows）

1.2.1 安装步骤

  1. 下载mysql安装包（推荐社区版，官网或国内镜像），选择自定义安装，勾选mysql server、mysql workbench（可视化工具）。

  2. 配置安装路径（建议非c盘，如d:\mysql\mysql server 8.0），避免中文路径。

  3. 配置服务器：选择“独立服务器”模式，端口默认3306（可修改，避免冲突），设置root用户密码（牢记，后续登录用）。

  4. 安装完成后，启动mysql服务：控制面板→服务→找到mysql，设置“自动启动”，确保服务正常运行。

1.2.2 登录mysql（两种方式）

  1. 命令行登录（推荐，适合博客演示操作）：
    `mysql -u root -p

# 输入设置的root密码，回车即可登录成功`

  1. 可视化工具登录（mysql workbench）：打开工具，新建连接，输入主机（localhost）、端口（3306）、用户名（root）、密码，点击连接即可。

1.2.3 常见问题解决

  • 服务启动失败：检查端口是否被占用（用netstat -ano | findstr 3306查看），或重新配置mysql服务。

  • 登录失败：确认密码正确，若忘记密码，可通过重置密码脚本（mysqld --skip-grant-tables）重置。

1.3 数据库基础概念

  • 数据库（database）：是一个存储数据的容器，本质是一个或多个文件的集合，用于组织和管理数据（如：blog_db、user_db）。

  • 表（table）：数据库中的基本存储单元，用于存储具体的数据，每个表有唯一的名称，由行（记录）和列（字段）组成。

  • 字段（field）：表中的一列，用于描述数据的某个属性（如：user表中的id、name、age字段），每个字段有对应的数据类型。

  • 记录（row）：表中的一行，对应一条具体的数据（如：user表中id=1、name=张三、age=20，就是一条记录）。

  • sql：结构化查询语言，用于操作mysql数据库的标准语言，分为ddl、dml、dql、dcl四类（后续详细讲解）。

二、sql基础（核心）

sql（structured query language）是操作关系型数据库的标准语言，所有关系型数据库（mysql、oracle、sql server）都支持sql，语法基本一致，重点掌握mysql的专属语法细节，以下所有sql语句均使用小写。

三、mysql事务（基础部分完结）

3.1 事务的定义

事务（transaction）是一组不可分割的sql操作集合，要么全部执行成功，要么全部执行失败，不会出现部分执行成功、部分失败的情况，用于保证数据的完整性和一致性。

举个例子：转账操作（从a账户转100元到b账户），包含两个sql操作：

  1. a账户余额减100元（update）

  2. b账户余额加100元（update）

这两个操作必须同时成功或同时失败，否则会出现“钱少了”或“钱多了”的问题，这就是事务的作用。

3.2 事务的四大特性（acid，核心，面试常考）

  • 原子性（atomicity）：事务是一个不可分割的整体，要么全部执行，要么全部回滚（撤销），不会出现部分执行的情况。

  • 一致性（consistency）：事务执行前后，数据的完整性约束不变。例如：转账前a+b的余额=转账后a+b的余额。

  • 隔离性（isolation）：多个事务同时执行时，彼此之间不会相互干扰，每个事务都感觉不到其他事务的存在（后续会讲隔离级别）。

  • 持久性（durability）：事务执行成功后，数据会永久保存到数据库中，即使数据库崩溃，数据也不会丢失（会写入磁盘）。

3.4 事务的隔离级别（基础认知）

mysql默认的事务隔离级别是“可重复读（repeatable read）”，用于解决多个事务同时执行时出现的并发问题（脏读、不可重复读、幻读），基础阶段只需了解隔离级别的作用，无需深入复杂配置。

四种隔离级别（从低到高）：

  1. 读未提交（read uncommitted）：最低级别，允许读取未提交的事务数据，会出现脏读、不可重复读、幻读。

  2. 读已提交（read committed）：允许读取已提交的事务数据，解决脏读，仍会出现不可重复读、幻读。

  3. 可重复读（repeatable read）：mysql默认级别，保证同一事务中多次读取同一数据结果一致，解决脏读、不可重复读，仍会出现幻读（mysql已通过mvcc机制解决幻读）。

  4. 串行化（serializable）：最高级别，事务串行执行，完全解决并发问题，但效率极低，适合数据一致性要求极高的场景（如金融转账）。

3.5 事务的注意事项

  • 事务只能用于dml语句（insert、update、delete），ddl语句（create、drop、alter）会自动提交事务，无法回滚。

  • 手动控制事务时，必须先执行start transaction，再执行dml语句，最后用commit或rollback结束。

  • 事务开启后，若未执行commit或rollback，关闭mysql连接，事务会自动回滚。

  • 避免长时间开启事务，会占用数据库资源，影响其他操作的执行效率。
