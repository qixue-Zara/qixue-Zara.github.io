PHP个性化难点分析

PHP学习笔记（难点解析）

笔记一：个性化难点解析（菜鸟教程学习重点+难懂点纠正）

本笔记聚焦学习过程中容易混淆、理解不透彻的知识点，结合自身疑问纠正偏差，重点突破个性化难点，适配Typora阅读和编辑，补充所有遗漏的核心难点，确保覆盖PHP全场景易踩坑点。

一、变量作用域（核心难点）

核心结论：函数内无法直接访问全局变量，函数外也无法直接访问函数内的局部变量，需通过指定方式调用；静态变量是特殊的局部变量，需重点区分。

  • 全局变量调用：函数内访问全局变量，必须使用 global 关键字，或通过 $GLOBALS\[\&\#39;变量名\&\#39;\] 访问（两者等价，重点理解后者）。

    • 补充：global $x 本质是引用全局变量$x，与 $GLOBALS\[\&\#39;x\&\#39;\] 操作的是同一个变量，无区别。

  • 关键解析（纠正理解偏差）：
    对于 $GLOBALS\[\&\#39;y\&\#39;\] = $GLOBALS\[\&\#39;x\&\#39;\] \+ $GLOBALS\[\&\#39;y\&\#39;\]; 这句话，三个y是同一个全局变量，并非三个不同变量：

    等价于简化写法：$y = $x \+ $y;，逻辑完全一致，只是前者明确指定操作全局变量（避免与局部变量混淆）。

    1. 右边的 $GLOBALS\[\&\#39;y\&\#39;\]：读取全局变量y的原始值；

    2. 左边的 $GLOBALS\[\&\#39;y\&\#39;\]：将计算后的新值，重新赋值给全局变量y；

    3. 后续 echo $y：输出的是被重新赋值后的全局变量y。

  • 局部变量限制：函数内定义的局部变量，仅在函数内部有效，函数执行结束后自动销毁，外部无法直接调用，只能通过调用函数间接使用其结果。

  • 补充难点：静态变量（static）

    • 定义：在函数内用 static $变量名 = 初始值; 定义；

    • 核心特性：函数执行结束后，变量值不销毁，保留上次执行的结果，下次调用函数时直接使用该值；

    • 易错点：静态变量仅在第一次调用函数时初始化，后续调用不再重新赋值。

二、Heredoc（定界符）语法（易踩坑点）

Heredoc用于定义多行字符串，语法有严格要求，稍不注意就会报错，重点记3点，补充遗漏细节：

  1. 标记规则：以 \&lt;\&lt;\&lt;EOF 开始（EOF可替换为EOT、EOD、JSON、HTML等，自定义即可），以 EOF; 结束；

    • 补充：开始标记后的EOF（或自定义标记），后面不能加任何字符（包括空格、注释），否则报错。

  2. 结束标记要求：必须顶头写，不能有任何缩进、空格，且末尾必须加分号；

    • 易错点：结束标记前后不能有任何多余字符，哪怕是一个空格，都会导致解析失败。

  3. 解析规则：标记之间的变量可正常解析（无需用 \. 或 , 拼接），但函数无法解析；

    • 补充：若需在Heredoc中显示变量名（不解析），可在变量前加反斜杠 \\$变量名；

    • 补充：Nowdoc语法（补充遗漏）：与Heredoc类似，标记用单引号包裹（如 \&lt;\&lt;\&lt;\&\#39;EOF\&\#39;），标记之间的变量、转义字符都不解析，适合定义纯文本。

三、var_dump()函数（易误用点）

核心注意：var\_dump\(\) 是自带输出功能的函数，无需搭配 echo，调用后会直接将变量的数据类型和值打印到页面，无需额外写输出语句。

  • 补充细节：

    • 可同时传入多个变量，用逗号分隔，如 var\_dump\($a, $b, $c\);；

    • 与 print\_r\(\) 区别（补充遗漏）：print\_r\(\) 仅输出变量的值（数组会格式化显示），不输出数据类型；var\_dump\(\) 输出数据类型+值，更适合调试。

四、PHP常量（易与变量混淆）

常量与变量的核心区别和关键特性，纠正易混淆点，补充遗漏细节：

  1. 不变性：一旦定义，值无法修改、无法删除，也无法重新定义；

  2. 作用域：全局作用域，定义后可在脚本任何地方使用（包括函数内、类内），无需用global关键字；

    • 补充：类常量（补充遗漏）：在类内用 const 常量名 = 常量值; 定义，只能通过 类名::常量名 或对象名::常量名 访问，不能用$this访问。

  3. 语法规范：名称无需加 $ 修饰符，由字母、下划线、数字组成（数字不能开头）；

    • 补充：常量名建议全部大写，便于区分（规范写法，非强制）。

  4. 大小写：默认区分大小写，define()函数第三个参数设为true，可定义大小写不敏感的常量；

    • 补充：const定义的常量，默认区分大小写，且不能设置为不敏感；define()可设置，二者定义方式的区别见下文。

  5. 值的要求：只能是标量类型（布尔值、整数、浮点数、字符串）或PHP7及以上版本的数组，不能是变量、类属性、数学运算的结果或函数调用；

  6. 补充：define()与const的区别（遗漏重点）：

    • 定义位置：const可在类内、类外定义；define()只能在类外定义；

    • 解析时间：const在编译时解析；define()在运行时解析；

    • 灵活性：define()可通过变量动态定义常量名（如 $name = \&\#39;PI\&\#39;; define\($name, 3\.14\);）；const不行。

五、字符串拼接（易出错点）

PHP中字符串拼接使用 \.（并置运算符），而非 \+（+用于数值运算），重点记两种用法，补充遗漏细节：

  1. 普通拼接：$b = $a \. \&\#34; world\!\&\#34;;（拼接变量和字符串，中间可加空格）；

    • 易错点：若拼接的变量为null、false，会被转为空字符串；若为数字，会被转为字符串后拼接。

  2. 复合赋值拼接：$x \.= \&\#34; world\!\&\#34;;（等价于 $x = $x \. \&\#34; world\!\&\#34;;，简化写法）。

六、运算符（难点：取反、异或、优先级，补充遗漏运算符）

1. 重点单目运算符（易混淆）

运算符 作用 示例 输出结果
-x 取x的相反符号 $x=2; echo -$x; -2
~x 按二进制位取反（运算规则：~n = -n-1） $x=2; echo ~$x; -3
++x 前置递增：先加1，再使用 $x=2; echo ++$x; 3
x++ 后置递增：先使用，再加1 $x=2; echo $x++; 2

2. 异或运算符（xor）

核心逻辑：两个条件有且仅有一个为true，返回true；否则返回false（同真同假均为false）。

3. 补充遗漏运算符（重点）

  • 三元运算符（补充细节）：$a = 条件 ? 值1 : 值2;，条件为true返回值1，否则返回值2；

    • 易错点：三元运算符是右结合，如 $a = 1 ? 2 : 3 ? 4 : 5; 等价于 $a = 1 ? 2 : \(3 ? 4 : 5\);，结果为2。

  • 组合比较运算符（补充遗漏）：$a \&lt;=\&gt; $b，用于比较两个值，返回-1（a&lt;b）、0（a==b）、1（a&gt;b），适合排序场景；

  • 逻辑运算符补充：and 和 \&amp;&amp;、or 和 \|\| 功能相同，但优先级不同（\&amp;\&amp; \&gt; \|\| \&gt; and \&gt; or）；

    • 示例：$a = false and true; 等价于 \($a = false\) and true;，$a的值为false；

    • 示例：$a = false \&amp;\&amp; true; 等价于 $a = \(false \&amp;\&amp; true\);，$a的值为false。

4. 运算符优先级（关键，避免运算错误）

优先级从高到低（完整版本，重点记常用，无需死记全部，关键是用括号改变优先级）：

  1. 括号（()）：优先级最高，可强制改变运算顺序；

  2. 类型转换、递增/递减（如：(int)、(float)、(string)、++、--、~）；

  3. 算术运算符（*、/、% &gt; +、-、.）；

  4. 位运算符（&lt;&lt;、&gt;&gt; &gt; &amp; &gt; ^ &gt; |）；

  5. 比较运算符（==、!=、===、!==、&lt;&gt; &gt; &gt;、&lt;、&gt;=、&lt;= &gt; &lt;=&gt;）；

  6. 逻辑运算符（! &amp;gt; &amp;amp;&amp;amp; &amp;gt; ||）；

  7. 三元运算符（? :）；

  8. 赋值运算符（=、+=、-=、*=、/=、.=、%=、&amp;=、|= 等）；

  9. 逻辑运算符（and &gt; xor &gt; or）；

  10. 逗号运算符（,）：优先级最低，用于分隔多个表达式，返回最后一个表达式的值。

关键提醒：赋值运算符遵循“左操作数 = 左操作数 运算 右操作数”，如 $x \+= $y 等价于 $x = $x \+ $y；字符串拼接的复合赋值$x \.= $y 同理。

七、面向对象（核心难点，补充完整）

核心类比：类是“模板”（定义属性和方法），对象是“模板实例化后的具体个体”，与C语言结构体+操作函数等价，补充所有遗漏的面向对象知识点，重点记以下内容：

  1. $this：代表当前对象，只能在类的非静态方法内部使用，用于访问当前对象的属性和方法；

    • 易错点：$this 不能在静态方法中使用，因为静态方法不依赖具体对象。

  2. 构造函数：用于对象初始化，创建对象时自动执行，用于给对象属性赋值；

    • 写法：public function \_\_construct\(参数1, 参数2, \.\.\.\)\{ \.\.\. \}；

    • 补充：PHP7及以上支持构造函数属性提升（简化写法），如 public function \_\_construct\(public $name, public $age\)\{\}，无需单独定义属性。

  3. 析构函数（补充遗漏）：public function \_\_destruct\(\)\{ \.\.\. \}，对象被销毁时自动执行（脚本结束时或手动unset()对象），常用于释放资源（如关闭文件、数据库连接）。

  4. 静态（static）：

    • 静态属性/方法：无需实例化类，可直接通过 类名::属性名、类名::方法名\(\) 访问；

    • 注意：静态方法中不能使用 $this（无具体对象），静态属性不能通过对象-&gt;访问；

    • 补充：静态属性是所有对象共享的，修改一个对象的静态属性，会影响所有该类的对象。

  5. 访问控制修饰符（补充遗漏，核心重点）：

    • public（公共）：类内、类外、子类均可访问；

    • private（私有）：仅类内可访问，类外、子类均不可访问；

    • protected（受保护）：类内、子类可访问，类外不可访问；

    • 易错点：子类不能直接访问父类的private属性/方法，需通过父类的public/protected方法间接访问。

  6. 继承（补充细节）：

    • 用extends关键字，子类继承父类的public/protected属性和方法，private属性/方法无法继承；

    • 重写：子类可定义与父类同名的方法，重写时访问修饰符不能比父类更严格（如父类是public，子类不能是private）；

    • 父类调用：子类中用 parent::方法名\(\) 调用父类的方法，用 parent::\_\_construct\(\) 调用父类的构造函数。

  7. 抽象类与接口（补充遗漏，难点）：

    • 抽象类：用abstract关键字定义，不能实例化，只能被继承；类内可包含抽象方法（无方法体，用abstract定义）和普通方法；子类必须实现抽象类中所有的抽象方法；

    • 接口：用interface关键字定义，不能实例化，只能被实现（implements）；接口内只能定义抽象方法（PHP8.0+可定义常量和默认方法）；一个类可实现多个接口，用逗号分隔。

  8. 魔术方法（补充遗漏，常用）：

    • \_\_get\($name\)：访问不存在或不可访问的属性时自动调用；

    • \_\_set\($name, $value\)：给不存在或不可访问的属性赋值时自动调用；

    • \_\_call\($name, $args\)：调用不存在或不可访问的方法时自动调用；

    • \_\_toString\(\)：将对象转为字符串时自动调用（如echo对象），需返回字符串。

八、表单相关（易踩坑点，补充完整细节）

  1. 单选框（radio）：同一组选项的 name 属性必须相同，否则无法实现“单选”效果，提交的是 value 值，而非界面显示的文字；

    • 补充：若需默认选中某一个选项，给该选项添加 checked=\&\#34;checked\&\#34;（或简写为checked）。

  2. 复选框（checkbox，补充遗漏）：

    • 同一组选项的name属性需设为数组形式，如 name=\&\#34;hobby\[\]\&\#34;，否则只能获取最后一个选中的值；

    • 提交后，通过 $\_GET\[\&\#39;hobby\&\#39;\] 或 $\_POST\[\&\#39;hobby\&\#39;\] 获取，返回的是数组（选中的所有value值）。

  3. 文件上传：

    • 表单必须添加属性：enctype=\&\#34;multipart/form\-data\&\#34;，否则无法获取上传文件；

    • 表单提交方式必须为POST，GET方式有长度限制，无法传递文件数据；

    • 获取文件：通过 $\_FILES 超全局数组获取，核心参数：

      • $\_FILES\[\&\#34;file\&\#34;\]\[\&\#34;name\&\#34;\]：上传文件的原始名称；

      • $\_FILES\[\&\#34;file\&\#34;\]\[\&\#34;type\&\#34;\]：上传文件的MIME类型（如image/jpeg）；

      • $\_FILES\[\&\#34;file\&\#34;\]\[\&\#34;tmp\_name\&\#34;\]：文件上传到服务器后的临时路径（必须用此路径操作文件）；

      • $\_FILES\[\&\#34;file\&\#34;\]\[\&\#34;error\&\#34;\]：上传错误码（0表示成功，1表示文件超过php.ini限制，2表示文件超过表单限制等）；

      • $\_FILES\[\&\#34;file\&\#34;\]\[\&\#34;size\&\#34;\]：上传文件的大小（单位：字节）。

    • 关键判断：$\_FILES\[\&\#34;file\&\#34;\]\[\&\#34;error\&\#34;\] == 0 表示文件上传成功；

    • 常用操作：限制文件大小（$\_FILES\[\&\#34;file\&\#34;\]\[\&\#34;size\&\#34;\]）、获取文件后缀（explode \+ end 或 pathinfo\(\)）、保存文件（move\_uploaded\_file\($\_FILES\[\&\#34;file\&\#34;\]\[\&\#34;tmp\_name\&\#34;\], 目标路径\)）；

    • 易错点：保存文件时，目标路径必须有可写权限（可通过chmod命令修改），否则保存失败。

  4. 表单过滤：使用filter\_input\_array\(\) 过滤表单输入，第二个参数可指定过滤器，非法输入会返回false；

    • 补充：常用过滤器（遗漏）：

      • FILTER_VALIDATE_EMAIL：验证邮箱格式；

      • FILTER_VALIDATE_URL：验证URL格式；

      • FILTER_VALIDATE_INT：验证是否为整数；

      • FILTER_SANITIZE_STRING：过滤HTML标签，保留纯文本。

    • 示例：$filters = array\(\&\#39;email\&\#39; =\&gt; FILTER\_VALIDATE\_EMAIL, \&\#39;age\&\#39; =\&gt; FILTER\_VALIDATE\_INT\);，过滤后若email格式错误，$result[&#39;email&#39;]为false。

  5. 表单验证补充：除了过滤器，还需手动验证必填项（如 empty\($\_POST\[\&\#39;name\&\#39;\]\) 判断是否为空）、数据长度、数据格式等，避免非法数据提交。

九、会话（Session，补充遗漏，核心重点）

Session用于在多个页面之间保存用户信息（如登录状态），与Cookie的区别是：Session数据保存在服务器端，更安全；Cookie保存在客户端，易被篡改。

  • 核心操作：

    1. 启动会话：session\_start\(\);（必须放在&lt;html&gt;标签之前，且页面无任何输出，否则报错）；

    2. 设置Session：$\_SESSION\[\&\#39;key\&\#39;\] = value;（如 $\_SESSION\[\&\#39;username\&\#39;\] = \&\#39;admin\&\#39;;）；

    3. 获取Session：echo $\_SESSION\[\&\#39;key\&\#39;\];（需先启动会话）；

    4. 销毁Session：

      • 销毁单个Session：unset\($\_SESSION\[\&\#39;key\&\#39;\]\);；

      • 销毁所有Session：session\_destroy\(\);（需先启动会话，销毁后需手动清除$_SESSION数组）；

      • 补充：销毁Session后，用户会话结束，下次访问需重新启动会话。

  • 易错点：

    • 每个使用Session的页面，都必须先调用 session\_start\(\);，否则无法访问Session数据；

    • Session默认有效期为浏览器关闭后，若需设置长期有效，需修改php.ini中的session.cookie_lifetime参数，或手动设置Cookie有效期。

十、Cookie（补充遗漏细节）

  • 设置Cookie：setcookie\(\&\#34;name\&\#34;, \&\#34;value\&\#34;, 过期时间, 路径, 域名, 安全标志, HttpOnly标志\);，必须放在 \&lt;html\&gt; 标签之前；

    • 补充参数说明：

      • 过期时间：Unix时间戳（如 time\(\) \+ 3600 表示1小时后过期），0表示浏览器关闭后过期；

      • 路径：Cookie生效的路径（如 &#39;/&#39; 表示整个网站生效，&#39;/admin&#39; 表示仅admin目录生效）；

      • 域名：Cookie生效的域名（如 &#39;example.com&#39;）；

      • 安全标志：true表示仅在HTTPS协议下生效；

      • HttpOnly标志：true表示Cookie不能被JavaScript访问，防止XSS攻击。

  • 获取Cookie：$\_COOKIE\[\&\#34;name\&\#34;\]；

  • 删除Cookie：设置过期时间为过去的时间（如setcookie\(\&\#34;name\&\#34;, \&\#34;\&\#34;, time\(\) \- 3600\);）；

  • 易错点：

    • Cookie是客户端存储，用户可手动删除、修改，不能用于存储敏感信息（如密码）；

    • 若浏览器禁用Cookie，Session也无法正常使用（因为Session默认通过Cookie传递会话ID）。

十一、错误处理与异常处理（补充遗漏，重点）

1. 错误处理

  • 错误级别（常用，补充）：

    • E_ERROR：致命错误，脚本终止执行（如语法错误）；

    • E_WARNING：警告错误，脚本继续执行（如使用未定义变量）；

    • E_NOTICE：通知错误，脚本继续执行（如变量未初始化）；

    • E_PARSE：解析错误，脚本终止执行（如括号不匹配）。

  • 错误控制：

    • 抑制错误：在可能出错的函数前加 @ 符号，如 @fopen\(\&\#34;file\.txt\&\#34;, \&\#34;r\&\#34;\);，可抑制该函数的错误输出；

    • 设置错误级别：error\_reporting\(E\_ALL \&amp; \~E\_NOTICE\);，表示显示所有错误，排除通知错误；

    • 自定义错误处理函数：set\_error\_handler\(\&\#34;自定义函数名\&\#34;\);，可自定义错误的显示方式（如写入日志）。

2. 异常处理

异常是脚本执行过程中出现的可预测错误（如数据库连接失败），通过try...catch捕获，避免脚本终止执行。

  • 核心语法：
    `&lt;?php

try {
// 可能出现异常的代码
$conn = new PDO(&#34;mysql:host=localhost;dbname=test&#34;, &#34;root&#34;, &#34;123456&#34;);
if (!$conn) {
throw new Exception(&#34;数据库连接失败&#34;); // 手动抛出异常
}
} catch (Exception $e) {
// 捕获异常，处理错误
echo &#34;异常信息：&#34; . $e-&gt;getMessage(); // 获取异常信息
echo &#34;异常行号：&#34; . $e-&gt;getLine(); // 获取异常所在行号
}
?&gt;`

  • 补充：可自定义异常类（继承Exception类），用于区分不同类型的异常（如数据库异常、文件异常）。

十二、文件操作（补充遗漏，重点）

PHP可对服务器端文件进行读取、写入、删除等操作，需注意文件权限，常用操作如下：

  • 打开文件：fopen\($filename, $mode\)，返回文件资源句柄，$mode是打开模式（常用：r=只读、w=写入（覆盖原有内容）、a=追加写入、r+=读写、w+=读写（覆盖）、a+=读写（追加））；

  • 读取文件：

    • fread\($handle, $length\)：读取指定长度的内容；

    • fgets\($handle\)：读取一行内容；

    • file\_get\_contents\($filename\)：一次性读取整个文件内容（简化写法，无需打开/关闭文件）。

  • 写入文件：

    • fwrite\($handle, $content\)：将内容写入文件；

    • file\_put\_contents\($filename, $content\)：一次性写入整个文件内容（简化写法，无需打开/关闭文件）；

    • 追加写入：用a/a+模式打开文件，或 file\_put\_contents\($filename, $content, FILE\_APPEND\)。

  • 关闭文件：fclose\($handle\)，必须关闭文件资源，释放内存；

  • 其他常用操作：

    • file\_exists\($filename\)：判断文件/目录是否存在；

    • unlink\($filename\)：删除文件；

    • mkdir\($dirname\)：创建目录；

    • rmdir\($dirname\)：删除空目录；

    • copy\($source, $dest\)：复制文件。

  • 易错点：文件操作需确保服务器对目标文件/目录有相应权限（如写入权限），否则操作失败。

十三、其他难点（零散易忘点，补充完整）

  1. 命名空间（namespace）：用于区分同名的类、函数、常量，不同命名空间中的同名元素互不冲突；

    • 使用方式：用 namespace 命名空间名; 定义，调用时用 命名空间名\\类名 或 use 命名空间名\\类名;（简化调用）；

    • 全局代码需用 namespace \{ \.\.\. \} 包裹；

    • 补充：命名空间可嵌套，如 namespace MyProject\\Admin;，调用时用 MyProject\\Admin\\类名。

  2. 常用函数（补充遗漏）：

    • 页面重定向：header\(\&\#34;Location: 目标地址\&\#34;\);，必须放在\&lt;html\&gt; 标签之前，且重定向后需用 exit\(\) 终止脚本执行，避免后续代码执行；

    • 时间戳：time\(\)，返回当前时间的Unix时间戳（秒数）；date\($format, $timestamp\)，将时间戳转为指定格式（如 date\(\&\#34;Y\-m\-d H:i:s\&\#34;, time\(\)\) 输出当前时间）；

    • 终止脚本执行：exit\(\) 或 die\(\)（两者等价），执行后脚本后续代码不再执行；

    • 数组相关函数（补充）：

      • array\_push\($arr, $value\)：向数组末尾添加元素；

      • array\_pop\($arr\)：删除数组末尾的元素；

      • array\_shift\($arr\)：删除数组开头的元素；

      • array\_unshift\($arr, $value\)：向数组开头添加元素；

      • array\_merge\($arr1, $arr2\)：合并两个数组；

      • in\_array\($value, $arr\)：判断元素是否在数组中。

  3. 关联数组：

    • 两种创建方式：$age=array\(\&\#34;Peter\&\#34;=\&gt;\&\#34;35\&\#34;,\&\#34;Ben\&\#34;=\&gt;\&\#34;37\&\#34;\); 或 $age\[\&\#39;Peter\&\#39;\]=\&\#34;35\&\#34;;；

    • 访问方式：通过指定键名访问，如 echo $age\[\&\#39;Peter\&\#39;\];；

    • 补充：遍历关联数组常用foreach，如 foreach\($age as $key =\&gt; $value\)\{ echo $key \. \&\#34;:\&\#34; \. $value; \}。

  4. 超全局数组（补充完整，核心重点）：PHP内置的超全局数组，可在脚本任何地方访问（无需global关键字），常用的有：

    • $\_GET：获取GET方式提交的表单数据或URL参数；

    • $\_POST：获取POST方式提交的表单数据；

    • $\_FILES：获取文件上传相关数据；

    • $\_COOKIE：获取客户端的Cookie数据；

    • $\_SESSION：获取服务器端的Session数据；

    • $GLOBALS：包含所有全局变量的数组；

    • $\_SERVER：包含服务器相关信息（如请求头、脚本路径、客户端IP等）；

    • $\_REQUEST：包含$_GET、$_POST、$_COOKIE的数据（不推荐使用，安全性低）。

  5. 类型转换（补充遗漏，易混淆）：

    • 自动转换：PHP会根据上下文自动转换类型（如字符串和数字运算时，字符串转为数字）；

    • 强制转换：用 (类型) 语法，如 \(int\)$str（转为整数）、\(float\)$str（转为浮点数）、\(string\)$num（转为字符串）、\(array\)$obj（转为数组）；

    • 易错点：字符串转整数时，若字符串开头是数字，会转为对应的数字（如 &#34;123abc&#34; 转为123）；若开头不是数字，转为0。

笔记二：PHP完整知识点归纳（全面无遗漏，简洁易懂）

本笔记全面归纳菜鸟教程PHP全部核心知识点，补充所有遗漏内容，简洁明了、重点突出，适合日常复习和快速查阅，无冗余内容，覆盖PHP基础、进阶全场景。

一、基础入门

  1. PHP简介：服务器端脚本语言，可嵌入HTML，运行在服务器端，输出HTML到客户端，用于开发动态网站。

  2. PHP标记：\&lt;?php \.\.\. ?\&gt;（标准标记，必用）；\&lt;? \.\.\. ?\&gt;（短标记，需开启php.ini中的short_open_tag）；\&lt;% \.\.\. %\&gt;（ASP风格标记，不推荐）。

  3. 变量：

    • 无需声明，第一次赋值时自动创建，以 $ 开头，区分大小写（$a和$A是两个不同变量）；

    • 作用域：

      • 全局变量：函数外定义，函数内需用 global $变量名 或 $GLOBALS\[\&\#39;变量名\&\#39;\] 访问；

      • 局部变量：函数内定义，仅在函数内部有效，函数执行结束后自动销毁；

      • 静态变量：函数内用 static $变量名 定义，函数执行结束后值不销毁，保留上次执行结果。

    • 变量赋值：值赋值（默认，如 $a = $b，修改$a不影响$b）；引用赋值（如 $a = \&amp;$b，修改$a会影响$b）。

  4. 常量：

    • 定义：define\(\&\#34;常量名\&\#34;, 常量值, 大小写不敏感\)（类外可用）；const 常量名 = 常量值;（类内、类外均可，默认区分大小写）；

    • 特性：值不变、全局作用域（类常量除外）、无需$修饰符、区分大小写（可设置不敏感）；

    • 类常量：类内定义，通过 类名::常量名 访问，不能用$this访问。

  5. 注释：

    • 单行注释：// 注释内容 或 \# 注释内容；

    • 多行注释：/\* 注释内容 \*/（不能嵌套）。

二、数据类型与运算符

1. 数据类型

  • 标量类型（单一值）：

    • 布尔值（bool）：true（1）、false（0），区分大小写（仅true/false有效，TRUE/FALSE也可）；

    • 整数（int）：正负整数，支持十进制、八进制（0开头）、十六进制（0x开头）；

    • 浮点数（float/double）：带小数点的数或科学计数法（如3.14、1.2e3）；

    • 字符串（string）：单引号（不解析变量、转义字符仅\和&#39;有效）、双引号（解析变量、转义字符有效）、Heredoc（多行字符串，解析变量）、Nowdoc（多行字符串，不解析变量）。

  • 复合类型（多个值）：

    • 数组（array）：索引数组（默认从0开始）、关联数组（自定义键名）、多维数组（数组嵌套数组）
