Java 全面个性化学习笔记（含知识点总结+学习疑惑解答）

一、学习疑惑及解答

学习过程中，针对Java知识点的理解、操作细节等提出了相关疑问，解答整理如下，均为学习中的重点难点，助力巩固记忆。

  • 疑惑1：Java注释有几种？各自的用法和注意事项是什么？
    解答：Java注释共3种，核心区别的用法如下：

    • 单行注释（//）：仅注释一行，用于简单说明代码含义，//后内容均为注释，换行失效。

    • 多行注释（/* */）：可注释多行，用于说明代码块，不可嵌套使用，否则会导致注释异常。

    • 文档注释（/** */）：仅用于类、方法、接口，可配合javadoc工具生成API文档，支持@author、@param等标签，不能用于代码行中间。

  • 疑惑2：基本数据类型和引用数据类型的区别是什么？8种基本数据类型的取值范围和用法有哪些注意点？
    解答：核心区别：基本数据类型直接存储值，引用数据类型存储对象的地址（如String、数组）；

8种基本数据类型注意点：

- long类型末尾必须加L/l，否则默认视为int，超出int范围会报错；

- float类型末尾必须加F/f，否则默认视为double，虽可赋值但不规范；

- char类型用单引号包裹，可存储中文（Unicode编码），用双引号会视为String报错；

- boolean类型仅存true/false，不能用0/1代替，与C/C\+\+不同。
  • 疑惑3：三种变量类型（局部、成员、类变量）的区别的是什么？默认值规则是什么？
    解答：区别核心在定义位置、作用域和归属：

默认值规则：整数0、浮点0.0、char&#39;\u0000&#39;、boolean false、引用类型null。

- 局部变量：定义在方法/代码块中，作用域仅限当前区域，无默认值，必须初始化后使用；

- 成员变量：定义在类中、方法外，属于对象，作用域覆盖整个类，有默认值；

- 类变量：用static修饰，属于类，可通过类名直接访问，所有对象共享，有默认值。

  • 疑惑4：switch case语句的注意事项有哪些？容易出错的点是什么？
    解答：核心注意事项，也是易错点：

    • 表达式类型只能是byte、short、int、char、enum、String（Java7+）；

    • case后必须是常量，不能是变量，且不能重复；

    • break不能省略，否则会出现“case穿透”，执行当前case后继续执行后续所有case；

    • default可选，建议放最后，所有case不匹配时执行。

  • 疑惑5：String、StringBuffer的区别是什么？各自适合什么场景？
    解答：核心区别在“可变性”和效率：

    • String：不可变字符串，修改（拼接、替换）会创建新对象，适合不频繁修改的场景；

    • StringBuffer：可变字符串，修改不会创建新对象，线程安全但效率低，适合频繁修改字符串的场景（如循环拼接）。

  • 疑惑6：访问修饰符有4种，各自的访问权限范围是什么？
    解答：从高到低排序：

    • public：全项目可访问（当前类、同包、不同包子类、不同包非子类）；

    • protected：当前类、同包、不同包子类可访问，不同包非子类不可；

    • default（无关键字）：仅当前类、同包可访问；

    • private：仅当前类可访问，同包、子类均不可。

  • 疑惑7：Overload（重载）和Override（重写）的区别是什么？容易混淆的点是什么？
    解答：核心区别在“是否在同一类、方法签名是否改变”：

易错点：重写时参数列表必须完全一致，否则会变成重载。

- Overload（重载）：同一类中，方法名相同，参数列表（个数、类型、顺序）不同，返回值可不同，与修饰符无关；

- Override（重写）：子类与父类中，方法名、参数列表、返回值完全相同，子类修饰符权限不能低于父类，不能重写final方法。

  • 疑惑8：抽象类和接口的区别是什么？各自的使用场景是什么？
    解答：核心区别在“继承/实现、方法类型、成员变量”：

    • 抽象类：用abstract修饰，可继承（单继承），可包含抽象方法（无方法体）和普通方法，可有成员变量；适合定义有共性的类，子类需实现抽象方法。

    • 接口：用interface修饰，可实现（多实现），所有方法默认是public abstract（Java8前），无成员变量（只能有常量）；适合定义规范、实现多继承效果。

  • 疑惑9：集合框架中，ArrayList、LinkedList、HashSet、HashMap的区别和使用场景是什么？
    解答：分List和Set、Map两类区分：

    • ArrayList：List接口，数组实现，查询快、增删慢，适合频繁查询、少增删的场景；

    • LinkedList：List接口，链表实现，查询慢、增删快，适合频繁增删、少查询的场景；

    • HashSet：Set接口，哈希表实现，元素唯一、无序，无索引，适合去重、无需排序的场景；

    • HashMap：Map接口，哈希表实现，键值对存储，键唯一、值可重复，适合存储键值映射关系（如用户ID和用户信息）。

  • 疑惑10：Java 8新特性有哪些？核心重点是什么？
    解答：核心重点新特性，贴合学习常用场景：

    • Lambda表达式：简化匿名内部类写法，简化代码（如集合遍历）；

    • Stream API：简化集合、数组的操作（过滤、排序、映射等）；

    • 接口默认方法：接口中可定义带方法体的默认方法（用default修饰），无需所有实现类都实现；

    • 方法引用：简化Lambda表达式，如::语法，进一步精简代码。

二、Java 核心知识点总结

模块1：Java AI 编程助手

Java AI编程助手是辅助Java学习和开发的工具，依托人工智能技术，可高效解决学习和开发中的问题，提升效率，尤其适合新手排查疑惑、巩固知识点。

1.1 常用Java AI编程助手

  • IntelliJ IDEA 内置AI（CodeGeeX、Copilot）：集成在开发工具中，实时提示代码、补全语法、解释代码含义，可生成类、方法，适配开发场景。

  • 豆包AI：适合学习场景，可直接提问知识点、代码报错原因、逻辑疑问，给出详细解答和实例，贴合新手学习节奏。

  • ChatGPT：可生成代码、排查bug、优化代码结构，解释复杂知识点（如反射、多线程），适合进阶学习和实战开发。

1.2 学习使用建议（结合自身学习体验）

  • 新手阶段：不要过度依赖AI生成代码，先自主编写，遇到报错、逻辑不通时，用AI排查问题，理解错误原因，避免失去动手能力。

  • 知识点巩固：用AI提问易混淆知识点（如重载与重写、抽象类与接口），结合笔记对比记忆，加深理解。

  • 实战练习：用AI生成基础代码框架，自己补充逻辑、调试修改，提升实战能力，同时借助AI优化代码结构。

模块2：Java 基础语法

Java基础语法是学习的核心基石，所有后续知识点均基于此，需熟练掌握，避免基础错误，重点关注易错点和实操细节。

2.1 Java 注释

注释用于说明代码，不参与程序编译和运行，便于阅读和维护，分3种类型，细节如下：

  • 单行注释：// 注释内容，仅作用于一行，简洁高效，适合简单说明（如变量含义）。

实例：// 定义整数变量，存储年龄

  • 多行注释：/* 注释内容 */，可跨多行，适合说明代码块、类的作用，不可嵌套。

实例：
/\* \* 该方法用于计算两数之和 \* 接收两个int类型参数，返回求和结果 \*/

  • 文档注释：/** 注释内容 */，仅用于类、方法、接口，可配合javadoc工具生成API文档，支持多种标签。

实例：
/\*\* \* @author 自己的名字 \* @param a 第一个求和参数 \* @param b 第二个求和参数 \* @return int 两数之和 \*/

2.2 文档注释常用标签

标签规范文档注释结构，生成的API文档更清晰，常用标签及用法：

  • @author：标识类的作者，可多个（用逗号分隔），仅用于类注释。

  • @version：指定类的版本（如1.0.0），用于版本管理，仅用于类注释。

  • @param：说明方法参数，格式「@param 参数名 说明」，有多少参数写多少个。

  • @return：说明方法返回值，仅用于有返回值的方法，需明确返回值类型和含义。

  • @throws：说明方法可能抛出的异常，与方法声明中throws的异常对应。

  • 补充标签：@see（关联其他类/方法）、@since（标记引入的Java版本）。

API文档生成步骤：编写带文档注释的Java文件 → 打开CMD进入文件目录 → 执行命令javadoc \-d 输出目录 文件名\.java → 打开输出目录的index.html查看。

2.3 Java 对象和类

Java是面向对象编程语言，类是对象的模板，对象是类的实例，核心是“封装、继承、多态”。

  • 类的定义：用class关键字，包含成员变量（属性）和成员方法（行为）。

实例：
public class Student \{ // 成员变量（属性） String name; int age; // 成员方法（行为） public void show\(\) \{ System\.out\.println\(\&\#34;姓名：\&\#34; \+ name \+ \&\#34;，年龄：\&\#34; \+ age\); \} \}

  • 对象的创建：用new关键字，格式「类名 对象名 = new 类名();」，创建后通过「对象名.成员变量」「对象名.成员方法()」访问。

实例：Student student = new Student\(\); student\.name = \&\#34;张三\&\#34;; student\.show\(\);

  • 类的特性：封装（隐藏属性，通过getter/setter访问）、继承（子类继承父类属性和方法）、多态（同一方法有不同实现）。

2.4 Java 基本数据类型

Java数据类型分基本数据类型（8种）和引用数据类型（类、接口、数组等），基本数据类型直接存储值，具体分类及细节：

类型分类 具体类型 字节数 取值范围 注意事项
整数类型 byte 1 -128 ~ 127 常用作节省内存的小整数
short 2 -32768 ~ 32767 较少使用
int 4 -2³¹ ~ 2³¹-1（约±21亿） 最常用的整数类型
long 8 -2⁶³ ~ 2⁶³-1 末尾必须加L/l
浮点类型 float 4 单精度，精度较低 末尾必须加F/f
double 8 双精度，精度较高 最常用的浮点类型
字符类型 char 2 0 ~ 65535 单引号包裹，可存中文
布尔类型 boolean 无明确字节数 true/false 不可用0/1代替

2.5 Java 变量类型

变量是存储数据的容器，根据定义位置和作用域，分为3种类型，核心区别如下：

  • 局部变量

- 定义：方法、代码块（for、if）中

- 作用域：仅限当前方法/代码块

- 特点：无默认值，必须初始化后使用

- 实例：public void test\(\) \{ int num = 10; \}

  • 成员变量（实例变量）

- 定义：类中、方法外

- 作用域：整个类

- 特点：有默认值（按数据类型），属于对象，创建对象后访问

- 实例：public class Student \{ String name; \}

  • 类变量（静态变量）

- 定义：类中、方法外，用static修饰

- 作用域：整个类

- 特点：有默认值，属于类，无需创建对象，通过类名访问

- 实例：public class Student \{ static String school; \}

2.6 Java 变量命名规则

命名需遵循规则（避免报错）和规范（提升可读性），具体如下：

2.6.1 必须遵守的规则（报错风险）

  • 由字母（A-Z、a-z）、数字（0-9）、下划线（_）、美元符号（$）组成，不能以数字开头。

  • 不能使用Java关键字（class、int、if等）和保留字（goto、const）。

  • 区分大小写（name和Name是两个不同变量）。

2.6.2 推荐遵守的规范

  • 局部变量、成员变量：驼峰命名法（首字母小写，后续单词首字母大写），如studentName、userAge。

  • 类变量（静态变量）：全大写，单词间用下划线分隔，如SCHOOL_NAME、MAX_NUM。

  • 见名知意，避免无意义命名（如a、b、c），如存储密码用password，存储成绩用score。

2.6.3 错误实例（避免踩坑）

  • int 1num = 10;（以数字开头，报错）

  • String class = \&\#34;Java\&\#34;;（使用关键字class，报错）

  • double $score = 95\.5;（合法但不推荐）

2.7 Java 修饰符

修饰符用于修饰类、方法、变量，控制其访问权限、生命周期和行为，分访问修饰符和非访问修饰符两类。

2.7.1 访问修饰符（控制访问权限）

修饰符 当前类 同包 不同包子类 不同包非子类
public 可访问 可访问 可访问 可访问
protected 可访问 可访问 可访问 不可访问
default（无关键字） 可访问 可访问 不可访问 不可访问
private 可访问 不可访问 不可访问 不可访问

2.7.2 非访问修饰符（控制行为、生命周期）

  • static（静态）：修饰类变量、方法、代码块，属于类，无需创建对象访问；static方法不能访问非static成员。

  • final（最终）：修饰类（不能继承）、方法（不能重写）、变量（变为常量，赋值后不可修改）。

  • abstract（抽象）：修饰类（抽象类，不能实例化）、方法（抽象方法，无方法体，需子类实现）。

  • transient（瞬态）：修饰成员变量，序列化时会被忽略（后续序列化章节详细说明）。

2.8 Java 运算符

运算符用于对变量进行运算，分6大类，重点掌握常用运算符的用法和优先级，避免运算错误。

2.8.1 常用运算符分类及用法

  • 算术运算符：+（加）、-（减）、*（乘）、/（除）、%（取余）、++（自增）、--（自减）

- 注意：整数除法舍去小数部分；++/--前缀（先运算后使用）、后缀（先使用后运算）。

- 实例：int a=10, b=3; System\.out\.println\(a%b\); // 1

  • 赋值运算符：=、+=、-=、*=、/=、%=

- 简化赋值操作，如a \+= 5 等价于 a = a \+ 5。

  • 比较运算符：==、!=、&gt;、&lt;、&gt;=、&lt;=

- 返回boolean值（true/false）；==比较基本类型比“值”，比较引用类型比“地址”。

  • 逻辑运算符：&amp;（与）、|（或）、!（非）、&amp;&amp;（短路与）、||（短路或）

- 短路与/或效率更高（左边可判断结果时，右边不执行）。

- 实例：true \|\| false; // true（短路或，右边不执行）

  • 三元运算符：条件表达式 ? 表达式1 : 表达式2

- 简化if-else，条件为true执行表达式1，否则执行表达式2。
- 实例：int max = \(a\&gt;b\) ? a : b;（获取最大值）

  • 位运算符：&amp;、|、^、~、&lt;&lt;、&gt;&gt;（较少用，了解即可）。

2.8.2 运算符优先级（从高到低）

算术运算符 &gt; 比较运算符 &gt; 逻辑运算符 &gt; 赋值运算符（无需死记，用括号可改变优先级）。

2.9 Java 循环结构

循环结构用于重复执行一段代码，Java有3种循环，根据循环次数是否已知选择使用，核心掌握循环逻辑和控制方式。

  • for循环：已知循环次数，语法规范，结构清晰。

语法：for \(初始化表达式; 条件表达式; 更新表达式\) \{ 循环体; \}
实例（输出1-10）：
for \(int i=1; i\&lt;=10; i\+\+\) \{ System\.out\.println\(i\); \}

  • while循环：未知循环次数，仅知道循环条件。

语法：初始化表达式; while \(条件表达式\) \{ 循环体; 更新表达式; \}
实例（输出1-10）：
int i=1; while \(i\&lt;=10\) \{ System\.out\.println\(i\); i\+\+; \}
注意：条件为false时，循环体一次不执行。

  • do-while循环：未知循环次数，循环体至少执行一次。

语法：初始化表达式; do \{ 循环体; 更新表达式; \} while \(条件表达式\);
实例（输出1-10）：
int i=1; do \{ System\.out\.println\(i\); i\+\+; \} while \(i\&lt;=10\);

  • 循环控制：

- break：跳出当前循环，结束循环执行；

- continue：跳过当前循环剩余部分，进入下一次循环。

2.10 Java 条件语句

条件语句用于根据不同条件执行不同代码块，分if语句、if-else语句、if-else if-else语句，核心是逻辑判断。

  • if语句：单条件判断，条件为true执行代码块。

语法：if \(条件表达式\) \{ 代码块; \}
实例：if \(score \&gt;= 60\) \{ System\.out\.println\(\&\#34;合格\&\#34;\); \}

  • if-else语句：双条件判断，条件为true执行if块，否则执行else块。

语法：if \(条件表达式\) \{ 代码块1; \} else \{ 代码块2; \}

  • if-else if-else语句：多条件判断，依次判断条件，匹配则执行对应块，均不匹配执行else块。

实例（成绩分级）：
if \(score \&gt;= 90\) \{ System\.out\.println\(\&\#34;优秀\&\#34;\); \} else if \(score \&gt;= 80\) \{ System\.out\.println\(\&\#34;良好\&\#34;\); \} else if \(score \&gt;= 60\) \{ System\.out\.println\(\&\#34;合格\&\#34;\); \} else \{ System\.out\.println\(\&\#34;不合格\&\#34;\); \}

2.11 Java switch case

switch case用于多值判断，替代复杂的if-else if-else，适用于变量有多个固定值的场景，重点注意break的使用。

  • 语法：
    `switch (表达式) {
    case 值1:
    代码块1;
    break; // 跳出switch
    case 值2:
    代码块2;
    break;
    default:
    代码块n; // 所有case不匹配时执行
    break;

}`

  • 实例（判断星期）：
    `int week = 3;

switch (week) {
case 1: System.out.println(&#34;星期一&#34;); break;
case 2: System.out.println(&#34;星期二&#34;); break;
case 3: System.out.println(&#34;星期三&#34;); break;
default: System.out.println(&#34;输入错误&#34;); break;
}`

  • 注意事项（易错点）：

- 表达式类型：byte、short、int、char、enum、String（Java7+）；

- case后必须是常量，不能重复；

- break不能省略，否则出现“case穿透”；

- default可选，建议放最后。

2.12 Java Number &amp; Math 类

Number类是基本数据类型包装类的父类（抽象类），Math类是数学工具类，无需创建对象，直接通过类名调用方法。

2.12.1 Number类（包装类）

用于将基本数据类型转换为引用数据类型，方便在集合中使用（集合仅存引用类型），对应关系如下：

  • 核心功能（以Integer为例）：

- 自动装箱/拆箱（Java5+）：Integer num = 10; int n = num;（自动装箱/拆箱）；

- 字符串与基本类型转换：int num = Integer\.parseInt\(\&\#34;123\&\#34;\); String str = Integer\.toString\(123\);。

2.12.2 Math类（常用静态方法）

  • Math.abs(x)：返回x的绝对值（如Math.abs(-5) → 5）；

  • Math.max(x,y)、Math.min(x,y)：返回最大值、最小值；

  • Math.random()：返回0.0（含）~1.0（不含）随机数；

  • Math.round(x)：四舍五入取整（如Math.round(95.5) → 96）；

  • Math.pow(x,y)：返回x的y次方（如Math.pow(2,3) → 8.0）；

  • Math.sqrt(x)：返回x的平方根（如Math.sqrt(16) → 4.0）。

2.13 Java Character 类

Character类是char类型的包装类，提供大量静态方法，用于判断、转换字符，简化字符操作。

  • 常用静态方法：

- Character.isLetter(c)：判断是否为字母；

- Character.isDigit(c)：判断是否为数字；

- Character.isUpperCase(c)、Character.isLowerCase(c)：判断大小写；

- Character.toUpperCase(c)、Character.toLowerCase(c)：转换大小写。

  • 实例（判断字符类型）：
    `char c = &#39;5&#39;;

if (Character.isDigit(c)) {
System.out.println(&#34;数字&#34;);
} else if (Character.isLetter(c)) {
System.out.println(&#34;字母&#34;);
} else {
System.out.println(&#34;其他符号&#34;);
}`

2.14 Java String 类

String类用于表示字符串，是最常用的引用数据类型，核心特点：字符串不可变（创建后内容不能修改），所有字符串常量存储在字符串常量池。

  • 创建方式（2种）：

- 直接赋值（推荐）：String str1 = \&\#34;Java\&\#34;;（存储在常量池）；

- new关键字创建：String str2 = new String\(\&\#34;Java\&\#34;\);（存储在堆内存）；

- 区别：str1 == str2（false，地址不同），str1\.equals\(str2\)（true，内容相同）。

  • 常用方法（重点）：

- length()：返回字符串长度；

- equals(str)：判断内容是否相同（区分大小写）；

- equalsIgnoreCase(str)：判断内容是否相同（不区分大小写）；

- concat(str)：拼接字符串（等价于+）；

- charAt(index)：返回指定索引的字符（索引从0开始）；

- substring(start, end)：截取字符串（不含end索引）；

- trim()：去除前后空格（不除中间）；

- replace(old, new)：替换字符串；
- split(regex)：根据正则拆分字符串，返回数组。

  • 注意：所有修改字符串的方法（concat、replace等），都会创建新字符串，原字符串不变。

2.15 Java StringBuffer

StringBuffer类用于表示可变字符串，解决String类不可变的问题，适合频繁修改字符串的场景（如循环拼接）。

  • 核心特点：可变、线程安全（效率较低），修改字符串时不会创建新对象。

  • 常用方法：

- append(str)：拼接字符串（核心方法）；

- insert(index, str)：在指定索引插入字符串；

- delete(start, end)：删除指定范围的字符串；

- reverse()：反转字符串。

  • 实例：
    `StringBuffer sb = new StringBuffer();

sb.append(&#34;Java&#34;);
sb.append(&#34;教程&#34;);
System.out.println(sb.toString()); // Java教程`

2.16 Java 数组

数组是存储多个相同类型数据的容器，长度固定，可存储基本数据类型和引用数据类型，核心掌握数组的创建、初始化和遍历。

  • 数组的创建（3种方式）：

- 方式1：int\[\] arr = new int\[5\];（指定长度，默认初始化）；

- 方式2：int\[\] arr = new int\[\]\{1,2,3,4,5\};（指定元素，长度自动匹配）；

- 方式3：int\[\] arr = \{1,2,3,4,5\};（简化写法，最常用）。

  • 数组的遍历（3种方式）：

- for循环：for \(int i=0; i\&lt;arr\.length; i\+\+\) \{ System\.out\.println\(arr\[i\]\); \}；

- 增强for循环（foreach）：for \(int num : arr\) \{ System\.out\.println\(num\); \}；

- Arrays.toString(arr)：将数组转为字符串（需导入java.util.Arrays）。

  • 二维数组：数组的数组，创建方式：int\[\]\[\] arr = \{\{1,2\},\{3,4\},\{5,6\}\};，遍历需嵌套循环。

  • 注意：数组长度固定，不能动态增减；数组索引从0开始，越界会报错（ArrayIndexOutOfBoundsException）。

2.17 Java 日期时间

Java提供日期时间相关类，用于处理日期、时间的获取、转换和格式化，重点掌握Java 8新增的LocalDateTime类（替代旧API）。

2.18 Java 正则表达式

正则表达式是用于匹配字符串的模式，可用于验证、查找、替换字符串，Java中通过Pattern和Matcher类实现。

  • 核心元字符（常用）：

- .：匹配任意单个字符（除换行）；

- *：匹配前面的字符0次或多次；

- +：匹配前面的字符1次或多次；

- ?：匹配前面的字符0次或1次；

- []：匹配括号内的任意一个字符（如[0-9]匹配数字）；

- ^：匹配字符串开头；

- $：匹配字符串结尾；

- \d：匹配数字（等价于[0-9]）；

- \w：匹配字母、数字、下划线（等价于[a-zA-Z0-9_]）。

  • 常用场景（实例）：

- 验证手机号：String regex = \&\#34;^1\[3\-9\]\\\\d\{9\}$\&\#34;;；

- 验证邮箱：String regex = \&\#34;^\[a\-zA\-Z0\-9\_\-\]\+@\[a\-zA\-Z0\-9\_\-\]\+\(\\\\\.\[a\-zA\-Z0\-9\_\-\]\+\)\+$\&\#34;;；

- 验证身份证（18位）：String regex = \&\#34;^\[1\-9\]\\\\d\{5\}\(19\|20\)\\\\d\{2\}\(\(0\[1\-9\]\)\|\(1\[0\-2\]\)\)\(\(\[0\-2\]\[1\-9\]\)\|10\|20\|30\|31\)\\\\d\{3\}\[0\-9Xx\]$\&\#34;;。

  • Java中使用：
    `// 方式1：使用String的matches方法（简单场景）

boolean isPhone = &#34;13800138000&#34;.matches(&#34;^1[3-9]\\d{9}$&#34;);
// 方式2：使用Pattern和Matcher（复杂场景）
Pattern pattern = Pattern.compile(&#34;^1[3-9]\\d{9}$&#34;);
Matcher matcher = pattern.matcher(&#34;13800138000&#34;);
boolean isPhone = matcher.matches();`

2.19 Java 方法

方法是一段可重复执行的代码块，用于实现特定功能，提高代码复用性和可读性，核心掌握方法的定义、调用和重载。

  • 方法的定义语法：
    修饰符 返回值类型 方法名\(参数列表\) \{ 方法体; return 返回值; // 无返回值（void）时可省略 \}
    - 修饰符：可选（如public、static）；

- 返回值类型：无返回值写void，有返回值写对应数据类型；

- 参数列表：可选，多个参数用逗号分隔。

  • 方法的调用：

- 无返回值：方法名\(参数\);；

- 有返回值：返回值类型 变量名 = 方法名\(参数\);。

  • 方法的重载（Overload）：

- 定义：同一类中，方法名相同，参数列表（个数、类型、顺序）不同，返回值可不同；

- 实例：
public int add\(int a, int b\) \{ return a\+b; \} public double add\(double a, double b\) \{ return a\+b; \}

  • 方法的参数传递：

- 基本数据类型：值传递（修改形参不影响实参）；

- 引用数据类型：引用传递（修改形参影响实参，因为传递的是地址）。

2.20 Java 构造方法

构造方法是类的特殊方法，用于创建对象时初始化对象的成员变量，与类名相同，无返回值，无需手动调用（new对象时自动调用）。

  • 构造方法的特点：

- 方法名与类名完全相同；

- 无返回值（不能写void）；

- 若未手动定义，编译器会自动生成无参构造方法；

- 可重载（多个构造方法，参数列表不同）。

  • 实例：
    public class Student \{ String name; int age; // 无参构造 public Student\(\) \{\} // 有参构造（重载） public Student\(String name, int age\) \{ this\.name = name; this\.age = age; \} \} // 调用构造方法创建对象 Student s1 = new Student\(\); // 调用无参构造 Student s2 = new Student\(\&\#34;张三\&\#34;, 18\); // 调用有参构造

  • 注意：手动定义有参构造后，编译器不会再生成无参构造，需手动添加（避免报错）。
